1そもそも何をする仕組み？

3秒で説明：

「AIが作った文章を、自動でGoogleに投稿する前に、自分が🆗するか🆖するか決める仕組み」です。

これがないと…

  • ❌ AIが間違って「営業時間11時〜25時」と書いたまま投稿してしまう
  • ❌ AIが低評価の口コミに不適切な返信をして炎上する
  • ❌ 顧客に「なんでこんな投稿してるんですか」と怒られる

承認フローがあれば「人がOKしたものだけ」がGoogleに出ます。 = 100%安全。

⚠️ 重要 承認しないとGoogleには何も投稿されません。「自動運用」なのに承認しないと「停止」と同じです。1日1回は必ず処理してください。

2毎日の作業（5分）

パターンは2つ。どっちか好きな方を選べばOK。両方併用も可。

📱 パターンA: スマホでDiscordを見る（外出中ベスト）

Discordアプリを開く

「アカネ」というチャンネルを開いてください（スマホでもPCでも同じ）。

💡 Discordインストールがまだなら iPhone/AndroidのApp Storeで「Discord」検索してインストール → 既存ログイン情報でサインイン。

承認待ちメッセージを探す

こんな見た目のメッセージが流れています：

# アカネ　|　承認待ち
アカネ Bot
⚠️ 承認待ち 📝 (post) ID: apr-xxxxx
タイトル: [ミセナビ 投稿] サンプル カフェ
春のメニュー、桜のラテ（580円）登場！
自家焙煎エスプレッソに桜の風味を加えた一杯。
営業：10:00-22:00 / 大阪市中央区...
要請元: misenavi / 緊急度: medium
✅ 承認
❌ 却下
⏸ 保留

3つのボタンのどれかを押すだけ

ボタン意味こんな時に押す
✅ 承認これで投稿OK内容に問題なし
❌ 却下これは投稿しない事実誤認・不適切・季節外れ
⏸ 保留あとで判断調べてから決めたい / 時間ない

5分以内にGoogleに自動投稿される

承認したものは、最大5分以内にGoogleビジネスプロフィールへ自動配信されます。何もする必要なし。

✅ これで完了 Discordで承認 → 5分以内に自動でGoogleに投稿。これだけ。

💻 パターンB: Macのターミナルで一気に処理

ターミナルを開く

Macの「アプリケーション → ユーティリティ → ターミナル」を起動。または⌘+Spaceで「ターミナル」と入力してEnter。

このコマンドをコピペ

cd ~/misenavi-system && npm run approve

→ 承認待ちが1件ずつ表示されます。

1件ずつキー入力で処理

━━━ 承認待ち 3件 ━━━

[1] 📝 投稿 post-xxx
サンプル カフェ (store_001) 5分前
春のメニュー、桜のラテ（580円）登場！...

[1/3] 操作 (o/r/e/s/q): _
キー意味
o + Enter承認（OK）
r + Enter却下（Reject）
e + Enter修正（Edit）→ 新本文を入力
s + Enterスキップ（後回し）
q + Enter終了

最後に「今すぐ配信？」と聞かれる

承認したものを今すぐGoogleに送るかの確認。yを押すと即配信。何もしなくても5分以内のcronで送られます。

✅ 完了！ 終了したらターミナルを閉じてOK。

4ターミナル（CLI）の使い方

「ターミナル」は黒い画面のことです。Macの中にもとから入ってるアプリ。怖くないです。

基本コマンド一覧

📋 承認待ちを見る

npm run approve list

一覧だけ表示（操作しない）

🎯 1件ずつ処理（推奨）

npm run approve

対話モード。これだけ覚えればOK

📊 統計を見る

npm run approve stats

承認待ち何件、却下何件、配信済み何件

🚀 今すぐ配信

npm run approve deliver

承認済みを5分待たずに即送信

📝 投稿だけ見る

npm run approve list post

口コミ返信を除外して投稿のみ

💬 口コミ返信だけ見る

npm run approve list review_reply

投稿を除外して口コミだけ

💡 これだけ覚えればOK npm run approve これ1個だけ。あとは画面の指示に従うだけ。

ID指定で個別操作（上級・通常使わない）

# 承認だけ npm run approve ok post-xxxxx-xxxx # 承認＋即配信 npm run approve ok post-xxxxx-xxxx --deliver # 却下（理由つき） npm run approve no post-xxxxx-xxxx 季節外れのため # 修正（本文を差し替えて再承認待ち） npm run approve edit post-xxxxx-xxxx "新しい投稿本文"

7困った時のFAQ

承認したのにGoogleに投稿されてない
承認後、最大5分以内に配信されます。それ以上待っても出ない場合：
npm run approve deliver で手動配信してみてください。
Discordの通知が来ない
pm2 logs lien-bot --lines 50 | grep misenavi でブリッジが起動しているか確認。
起動してなければ pm2 restart lien-bot で復旧。
ターミナルでコマンドが「command not found」と出る
ターミナルを開いた直後は cd ~/misenavi-system で先にディレクトリ移動してください。
これを忘れると npm が見つからずエラーになります。
承認待ちが大量に溜まってる
48時間で自動失効するので慌てなくてOK。一気に処理したい場合：
npm run approve で対話モード → o連打で全部承認も可。
間違って承認してしまった
5分以内なら間に合うかも。cd ~/misenavi-system/data && cat approval_pending.json でstatusが「approved」のまま「delivered」になっていなければ、status=rejectedに手動変更すればOK。
既にdeliveredならGoogleに投稿済み → GBP管理画面から手動削除してください。
「pending」「approved」「delivered」って何？
pending = 承認待ち（未処理）
approved = 承認済み（5分以内に配信される）
delivered = Googleに送信済み（完了）
rejected = 却下（送らない）
expired = 期限切れ（48時間放置で自動）
「ID」って何？覚える必要ある？
いいえ、覚える必要なし。対話モード（npm run approve）なら番号で操作できます。
「ID」は内部識別用です。エラー報告時だけ使います。
スマホだけで完結したい（Macは触りたくない）
DiscordだけでOK。ボタン押下のみで運用可能。
ただし通知見落としを防ぐため、Discordの通知をONに、ミュートにしないこと。

8緊急時の対処

🆘 「すべてが動いてない！」と感じた時

状態確認

pm2 list

misenaviとlien-botがonlineになっているか確認。

停止していたら再起動

pm2 restart misenavi && pm2 restart lien-bot

ログを見る

pm2 logs misenavi --lines 50 pm2 logs lien-bot --lines 50

ERRORの行があれば、その内容を担当者（竹林）にスクショで送ってください。

ヘルスチェック

cd ~/misenavi-system && npm run health

環境変数・GBPトークン・accounts.jsonをまとめて確認します。

🆘 「顧客から不適切な投稿が出てるとクレーム来た」

  1. Googleビジネスプロフィール管理画面 → 投稿 → 該当投稿を削除
  2. 口コミ返信の場合 → 該当口コミ → 返信を編集 or 削除
  3. 原因究明: cd ~/misenavi-system && cat data/approval_history.json | grep -A5 該当ID で承認履歴確認
  4. 同じことが起きないよう、contentGenerator.js のプロンプトを修正

🆘 「PCも触れない、Discordも見れない」

  1. 承認は48時間TTLで自動失効するので、放置でも顧客GBPには影響なし
  2. ただし投稿が止まるので、戻り次第まとめて承認
  3. 長期不在の場合は、別の人に「npm run approve」を任せる or APPROVAL_REQUIRED=falseで自動運用に切り替え
