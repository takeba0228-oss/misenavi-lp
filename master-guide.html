AIOとは

従来のSEOが「検索結果のリンクをクリック」を目指したのに対し、AIOは 「AIの回答文中に直接引用される」 ことを目指す。

AIOで重要な4要素

  1. 構造化データ（JSON-LD）: LocalBusiness/FAQPage 機械が理解しやすい形
  2. 一問一答型FAQ: 質問文をそのまま見出しに
  3. 第三者言及: 口コミ・引用される文脈の蓄積
  4. 明示的なエンティティ情報: 住所・営業時間・特徴の自然言語化

9困った時のFAQ

承認したのにGoogleに投稿されてない
承認後、最大5分以内に配信されます。それ以上待っても出ない場合は npm run approve deliver で手動配信。
Discord通知が来ない
pm2 logs lien-bot --lines 50 | grep misenavi でブリッジ起動確認。起動してなければ pm2 restart lien-bot。
ターミナルで「command not found」
先に cd ~/misenavi-system でディレクトリ移動。これを忘れるとnpmが見つからない。
承認待ちが大量に溜まってる
48時間で自動失効するので慌てなくてOK。一気に処理：npm run approve → o連打。
間違って承認してしまった
5分以内なら間に合うかも。data/approval_pending.jsonを編集してstatus=rejectedに。既にdeliveredならGBP管理画面から手動削除。
口コミに不適切な返信が出てしまった
GBP管理画面 → 該当口コミ → 返信を編集 or 削除。contentGenerator.js のプロンプトを修正して再発防止。
顧客から「投稿が止まってる」とクレーム
承認待ちが溜まっている可能性大。即 npm run approve で処理。今後は1日1回以上の承認を習慣化。
PM2のmisenaviがstoppedになってる
pm2 logs misenavi --lines 100 で原因確認。ほとんどの場合 pm2 restart misenavi で復旧。
GBP API HTTP 403エラーが出る
権限不足 or トークン期限切れ。GBP管理画面で「管理者」になっているか再確認。node src/setup.js でリフレッシュトークン再取得。
SerpApi 401エラー
APIキー未設定 or 期限切れ。.envのSERPAPI_KEY確認。月100検索の無料枠を超えていないかダッシュボードで確認。
月次レポートPDFが生成されない
手動実行: cd ~/misenavi-system && DEBUG=1 npm run report で詳細ログ確認。フォントエラーなら ~/Library/Fonts/GenShinGothic-Heavy.ttf 確認。
承認モードを一時停止したい（休暇中など）
.envに APPROVAL_REQUIRED=false を追加 → pm2 restart misenavi。AIが直接投稿モードに。戻すときは行を削除して再起動。
スマホだけで完結したい
DiscordだけでOK。ボタン押下のみで運用可能。Discordの通知をミュートにしないこと。
Macが落ちると全部止まる？
PM2のlaunchdで自動起動登録済みなので、Mac再起動でも自動復帰します。電源切れる→復旧後3分以内に再開。

10緊急時の対処

🆘 「全部動いてない！」

状態確認

pm2 list

misenavi / lien-bot がonlineか確認。

停止していたら再起動

pm2 restart misenavi && pm2 restart lien-bot

ログを見る

pm2 logs misenavi --lines 50 pm2 logs lien-bot --lines 50

ヘルスチェック

cd ~/misenavi-system && npm run health

🆘 「不適切な投稿が出たとクレーム」

  1. GBP管理画面 → 投稿 → 該当を削除
  2. 口コミ返信なら → 該当口コミ → 返信を編集or削除
  3. data/approval_history.json で承認履歴確認
  4. contentGenerator.js のプロンプトを修正
  5. 顧客に電話orLINEで謝罪 → 再発防止策を提示

🆘 「PCもDiscordも見られない」

  1. 承認は48時間TTLで自動失効するので、放置でもGBPに影響なし
  2. 長期不在の場合は別の人に npm run approve を任せる
  3. または APPROVAL_REQUIRED=false で自動運用に一時切替

🆘 「顧客のGBPアカウントから締め出された」

  1. 顧客に連絡 → 再度管理者として招待してもらう
  2. 顧客のオーナー権限は不変なので、招待し直せば即復旧可能
